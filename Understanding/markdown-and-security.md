## 📝 Lesson 10 & 11 — Markdown Language Guide and Making Your GitHub Much More Secure

This lesson focused on using the Markdown markup language to create organized, attractive, and well-structured documentation on GitHub (such as `README.md` files).

### 🔑 Key Points

- **What is Markdown**:
  - Lightweight markup language that converts plain text into formatted text
  - Standard used across GitHub for `README.md`, Issues, Pull Requests, and comments
- **Basic Formatting Syntax**:
  - Headings: `# Heading 1`, `## Heading 2`, `### Heading 3`
  - Text Formatting: `**bold**`, `*italic*`, `~~strikethrough~~`
  - Lists: Unordered lists with `-` or `*`, and ordered lists with `1. 2. 3.`
  - Links and Images: `[Link Text](URL)` and `![Image Alt Text](URL-to-image)`
  - Code Blocks: Inline code using backticks (`code`) or code blocks using triple backticks (` ```language ... ``` `)
  - Blockquotes and Tables: Blockquotes with `>` and structured tables with pipe bars `|`

### 💡 Concept

- **Clear Documentation**:
  - A great GitHub project requires great documentation
  - Markdown allows text styling without needing heavy visual editors (WYSIWYG)

---

### 🔑 Key Points

- **Account Protection**:
  - Using strong, unique passwords
  - Enabling mandatory **Two-Factor Authentication (2FA)** via authenticator apps (TOTP), security keys, or SMS
  - Safely storing **Recovery Codes**
- **Access Keys & Tokens**:
  - Configuring **SSH Keys** for secure terminal authentication
  - Using **Personal Access Tokens (PAT)** instead of account passwords on the command line
- **Code Security**:
  - Paying close attention to `.gitignore` so API keys, passwords, or `.env` files are never pushed to public repositories

---

## 🛠️ Best Practices

### 1. Markdown Documentation

- Keep `README.md` clean, legible, and organized with hierarchical headings

### 2. Credential Security

- Enable 2FA immediately under account settings (`Settings > Password and authentication`)
- Set up SSH Keys for terminal operations without exposing credentials

---

## 📌 Important Concepts

- Markdown is automatically rendered by GitHub on any file with a `.md` extension
- GitHub account security protects both your personal projects and contributions to organization repositories
