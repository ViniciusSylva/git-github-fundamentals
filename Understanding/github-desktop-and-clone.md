## ⚙️ Lessons 06 & 07 — GitHub Desktop (Linux) & Cloning Repositories

## 📥 Installing GitHub Desktop on Linux

This lesson focused on installing and using GitHub Desktop in a Linux environment.

### 🔑 Key Points

- GitHub Desktop is a graphical interface (GUI) for Git
- On Linux, installation may require:
  - Community versions
  - Manual installation using .deb packages
- Allows you to:
  - Manage repositories visually
  - Track changes without using the terminal
  - Perform commits, pushes, and pulls easily

### 💡 Concept

Git can be used in two ways:
- CLI (Command Line Interface)
- GUI (Graphical User Interface)

---

## 📂 Cloning a Repository

This lesson introduced how to copy a remote repository to your local machine.

### 🔑 Key Points

- Cloning creates a complete local copy of a repository
- Includes:
  - Files
  - Commit history
  - Branches
- It is the first step when working on existing projects

---

## 🛠️ Ways to Clone

### 1. Using Git (Terminal)

git clone <repository-url>

### 2. Using GitHub Desktop

- Click on "Code" in the repository
- Select "Open with GitHub Desktop"
- Choose the local folder
- The tool handles everything automatically

---

## 📌 Important Concepts

- After cloning:
  - You work locally on your machine
  - Changes can be sent using git push
- Cloning is done once per project
- Enables:
  - Collaboration
  - Version control
  - Offline work
